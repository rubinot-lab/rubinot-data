## Executive Summary

TibiaPal.com is a Tibia fansite created by Kusnier (Pawel Kusnierek), featuring calculators, planning tools, hunting guides, and reference material. The site is a static HTML/CSS/JS application served via Cloudflare. It has no authentication system, no user accounts, and stores state only in the browser (localStorage for cookie consent). All tools are client-side JavaScript calculators with no backend API.

The site has **29 discoverable pages** organized into 4 categories: General, Tools, Guides/References, and Calculators. The highest-priority feature for Rubinot is the **Charm Planner**, documented in extensive detail below.

## Charm Planner (DETAILED)

### Overview & Purpose

The Charm Planner allows players to plan their charm point allocation before spending points in-game. Charms in Tibia have a **3-stage upgrade system** (T1, T2, T3), and the planner tracks cumulative costs and the relationship between Major and Minor charm currencies.

URL: `https://tibiapal.com/charm_planner`
JS source: `https://tibiapal.com/scripts/charm_planner.js`
CSS source: `https://tibiapal.com/css/main.css`

### Core Mechanics

**Two currency types:**
- **Major Charm Points** - earned from bestiary completion, spent on Major charms
- **Minor Charm Points (Echoes)** - generated by upgrading Major charms, spent on Minor charms

**Key relationship:** Each Major charm stage upgrade generates Minor echoes:
- Stage 1 (T1): generates 50 minor echoes
- Stage 2 (T2): generates 100 minor echoes  
- Stage 3 (T3): generates 200 minor echoes
- Base minor echoes available: 100 (without any major upgrades)

**Each charm has 3 stages (T1, T2, T3)** with escalating costs and effects.

### UI/UX Flow

**Layout:**
- Top section: Instructions, charm point input, summary stats
- Tab bar: "Major Charms" | "Minor Charms" (tabbed view, starts on Major)
- Charm grid: 3 columns of charm cards, each showing: name, icon, stage, cost breakdown

**Interaction model:**
- **Left click** on charm icon: advance stage (0->1->2->3), calls `charm_pressed(charm_id)`
- **Right click** on charm icon: decrease stage (3->2->1->0), calls `charm_unpressed(charm_id)`
- **Mouse hover** on charm: shows description text, calls `description_on_hover(charm_id)`
- **Reset All** button: resets all charms to stage 0

**Input field:**
- Optional "Your Charm Points" text input (validates budget for Major charms)
- If set, prevents overspending Major points with error message

**Summary display (updated in real-time):**
- "Total Major charm points used: X"
- "Total Minor charm points used: X"
- "Total Minor charm points available: X"
- "Assigned Major charms: T1 Low blow, T2 Overpower, ..."
- "Assigned Minor charms: T1 Gut, T3 Scavenge, ..."
- "Charm Description: [hover text]"
- "Error Message: [validation errors]"

**Error handling:**
- Major: "Not enough Major charm points. You need X more charm points."
- Minor: "Not enough Minor charm points (echoes). You need X more minor charm points."
- Downgrade protection: "Unassign a Minor charm first, otherwise would result in negative minor points."

### Data Model

```typescript
// Major charm state: key = charm_id, value = current stage (0-3)
interface MajorCharmState {
  [charmId: string]: 0 | 1 | 2 | 3;
}

// Major charm costs: key = charm_id, value = [T1 cost, T2 cost, T3 cost]
// Costs are CUMULATIVE per stage (not incremental)
// Stage 1 costs array[0], stage 2 costs array[0]+array[1], stage 3 costs all three
interface MajorCharmCosts {
  [charmId: string]: [number, number, number];
}

// Minor charm costs are FLAT per tier (same for all minor charms)
// T1 = 100, T2 = 150, T3 = 225

// Minor echoes generated per major charm stage upgrade
// T1 = 50, T2 = 100, T3 = 200

// Starting minor echoes available: 100
```

**State tracking:**
- `total_major_charm_points` - running total of major points spent
- `total_minor_charm_points` - running total of minor echoes spent
- `total_minor_charm_points_available` - base(100) + sum of echoes gained from major upgrades - minor spent

**Cost accumulation for Major charms:**
- Upgrading to T1: costs `array[0]` (e.g., Low Blow T1 = 800)
- Upgrading to T2: costs `array[1]` additionally (e.g., Low Blow T2 adds 1200, total 2000)
- Upgrading to T3: costs `array[2]` additionally (e.g., Low Blow T3 adds 4000, total 6000)
- Downgrading refunds the last stage cost

### Implementation Notes for Rubinot

**What TibiaPal does well:**
- Simple, intuitive click-to-upgrade interaction
- Clear cost display per charm
- Real-time summary updates
- Good error messages for budget constraints
- Visual distinction between Major/Minor with tabs

**What Rubinot should improve:**
- Use React state management instead of global vars + DOM manipulation
- Add charm categorization (Offensive/Defensive/Utility/Elemental)
- Add dark/light theme support consistent with eve.rubinot.dev
- Make charm images responsive/scalable (SVG preferred, PNG fallback)
- Add keyboard navigation and accessibility
- Implement URL-based state (shareable builds)
- Add build presets ("Best PvE build", "Best profit build", etc.)
- Add total cost summary and "what you can afford" visualization
- Mobile-first responsive design (TibiaPal is desktop-focused)

## Other Tools & Calculators

### TibiaLootSplit (`/tibialootsplit`)
**Purpose:** Split party hunt loot evenly among players.
**Input:** Paste from Tibia's Party Hunt analyzer clipboard.
**Features:**
- Regular split mode
- Add extra expenses (TC or gold per player)
- Remove players from split
- History table (Date, Time, Players, Total Profit, Per-player profit)
- Results stored in localStorage

### Charms Damage Calculator (`/charm_calculator`)
**Purpose:** Compare Overpower/Overflux charm vs elemental charms.
**Inputs:** Charm type (Overpower/Overflux), max HP/mana, monster health, monster elemental sensitivity.
**Output:** Pre-mitigation damage comparison.

### Exercise Weapons Calculator (`/exercise`)
**Purpose:** Calculate exercise weapon requirements for target skill.
**Two modes:**
1. Target Skill: "How many weapons to reach skill X?"
2. Target Weapons: "What skill with Y weapons?"
**Inputs:** Vocation/skill type, current skill + loyalty, target, double event toggle, private dummy toggle, TC price threshold (13,900 gp).

### Training Calculator (`/offlinetraining`)
**Purpose:** Calculate offline/online training time to target skill.
**Two modes:**
1. Target Skill: "How long to reach skill X?"
2. Target Time: "What skill after X hours?"
**Inputs:** Training type (offline/online), vocation, current skill, loyalty bonus.

### Imbue Calculator (`/imbue`)
**Purpose:** Find most cost-effective imbuement method (gold tokens vs market items).
**Inputs:** Imbue type (Vampirism/Strike/Void), gold token value, item market prices.
**Output:** Cost comparison table showing buy/don't buy recommendations per tier.

### Weekly Task Calculator (`/weekly_tasks`)
**Purpose:** Calculate weekly task rewards by level and difficulty.
**Input:** Character level (supports up to 3499).

### Exp Share Calculator (`/expshare`)
**Purpose:** Calculate shared experience level range.
**Input:** Character level.
**Output:** Min/max level for party exp sharing.

### Stamina Calculator (`/stamina`)
**Purpose:** Calculate time to regenerate stamina.
**Inputs:** Current stamina, target stamina.

### Bless Calculator (`/bless`)
**Purpose:** Calculate blessing costs by level.
**Inputs:** Level, Henricus toggle, Enhanced blessings toggle, Twist of Fate toggle.

### Levelling Calculator (`/expcalculator`)
**Purpose:** Calculate experience points needed between levels.
**Inputs:** Starting level, target level, optional exp/hour, optional hours/day.
**Output:** Total exp needed, estimated time to target.

### Leech Calculator (`/leech`)
**Purpose:** Calculate life/mana leech amounts.
**Formula:** 100% leech on first monster, +10% per additional monster hit.
**Inputs:** Damage dealt, leech %, number of monsters hit.

## Data Visualizations & UX Patterns

### Visual Patterns Used
- **Tabbed navigation** for categorized content (vocations, charm types)
- **Data tables** with sorting for hunting spots, bosses, bestiary
- **Grid card layout** for charm icons with stage indicators
- **Real-time calculation** with no submit buttons (auto-calculate on input change)
- **Checkbox checklists** for bestiary/bosstiary progress tracking
- **Color-coded text** for status (gold for instructions, colored charm names in assignment list)

### Design System
- **Background:** Dark blue/navy with checkered pattern (`checkered` class)
- **Text:** Light gray (#CCC) on dark backgrounds
- **Accent:** Gold for highlights, blue (#1c4b9b) for active tabs
- **Sidebar:** Left navigation with icon + text items
- **Right sidebar:** Community/partner content (Discord, Kick, ads)
- **Mobile:** Hamburger menu toggle for navigation
- **Font:** Verdana, Arial, Times New Roman, sans-serif
- **Image style:** Pixel art (Tibia game sprites), 80px charm icons

### No Advanced Visualizations
TibiaPal does NOT use:
- Charts or graphs
- D3.js or similar
- Progress bars
- Heat maps
- Comparison visualizations
All data is presented in tables and text.

## Technical Architecture

- **Frontend:** Static HTML/CSS/JS (no framework, no build step)
- **Hosting:** Cloudflare (CDN + Cloudflare Insights beacon)
- **JavaScript:** Vanilla JS, no dependencies (no jQuery, no React, no libraries)
- **State:** Client-side only. LocalStorage for cookie consent and loot split history.
- **API:** None. All calculations are client-side.
- **Analytics:** Google Analytics (UA-164338286-3)
- **Ads:** Google AdSense (ca-pub-2936542406889680)
